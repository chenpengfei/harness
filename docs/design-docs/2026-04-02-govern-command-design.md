# /govern 命令设计文档

**日期**：2026-04-02（重新设计：2026-04-04）
**状态**：已批准

---

## 概述

`/govern` 命令是所有安装了 Harness 的工程的持续治理工具，包括 harness 仓库本身和所有目标工程。它通过四轮静默扫描检测仓库健康状态，扫描逻辑直接对应 **Agent 友好性四原则**，汇总为简报后逐项与用户协商处理，防止设计偏移和知识断层积累成"熵增"。

---

## 设计目标

- **对抗熵增**：定期识别结构缺失、文档不一致、合约缺失、模式漂移，阻止细小偏差积累
- **原则驱动**：四轮扫描直接映射 Agent 友好性四原则，不是任意检查清单
- **通用性**：同一份命令文件适用于 harness 仓库和所有目标工程，无需区分模式
- **人类掌舵**：每个问题均暂停询问用户，不自动批量修改
- **无副作用扫描**：扫描阶段不修改任何文件，只收集问题列表

## 架构：入口检查 + 四轮扫描 + 逐项协商

```
/govern
    │
    ▼
入口检查（.harness/ 目录存在性）
    │
    ▼
第一轮：结构完整性
  ├── E-K-C-F 四目录存在且含 .md 文件（原则 3：防止错误复合）
  ├── harness-config.json 字段完整（若存在）
  ├── Phase 文件完整性（若 INSTALL.md 存在）
  ├── INSTALL.md 引用完整性（若 INSTALL.md 存在）
  ├── 命令-设计-执行闭环（若 docs/design-docs/ 存在）
  └── CLAUDE.md 存在性
    │
    ▼
第二轮：文档与现实一致
  ├── CLAUDE.md 健康度（≤200 行、无占位符）（原则 2：文档与代码一致）
  ├── 设计文档索引覆盖（若 index.md 存在）
  ├── Phase 标题与 INSTALL.md 一致性（若 INSTALL.md 存在）
  ├── knowledge/architecture.md 实质性（若文件存在）
  └── 无内部硬链接
    │
    ▼
第三轮：合约与知识沉淀
  ├── do.md 三个审批检查点完整（若 do.md 存在）（原则 1：测试即合约）
  ├── ADR 目录有实质内容（若 decisions/ 存在）
  └── 近期 feat/refactor 提交有对应设计文档（git 环境可用时）
    │
    ▼
第四轮：设计模式统一
  ├── E-K-C-F 术语一致性（原则 4：设计模式统一）
  ├── 人类掌舵/Agent 执行表述
  ├── 安装后无依赖表述
  └── 旧命令名称引用（/audit 应更新为 /govern）
    │
    ▼
简报输出（统计数字）
    │
    ▼
逐项协商（优先级：合约缺失 → 结构缺失 → 文档不一致 → 模式不统一 → 需人工确认）
    │
    ▼
协商完成摘要
```

## 关键设计决策

### 决策 1：通用性优先于模式分支

原始设计将 `/govern` 限定为"仅适用于 harness 仓库本身"。重新设计的核心改变是：**移除模式检测，改为单一入口（`.harness/` 目录存在性），每项检查基于相关文件是否存在来决定是否执行**。

这样做的好处：
- 同一份命令文件安装到目标工程后直接可用
- harness 仓库和目标工程享有相同的治理质量
- 新增检查项时只需修改一处

### 决策 2：四轮对应 Agent 友好性四原则

原始设计的三轮（术语一致性、结构完整性、原则遵守）是按检查类型组织的。重新设计改为按 **Agent 友好性原则** 组织：

| 轮次 | 原则 | 核心关注 |
|------|------|---------|
| 第一轮 | 原则 3：防止错误复合 | 结构是否完整，基础不踏空 |
| 第二轮 | 原则 2：文档与代码一致 | 文档是否真实反映现状 |
| 第三轮 | 原则 1：测试即合约 | 工作流合约和决策记录是否存在 |
| 第四轮 | 原则 4：设计模式统一 | 术语和模式是否前后一致 |

### 决策 3：扫描阶段静默收集，不逐条输出

扫描期间只显示进度（`正在扫描…（第 N/4 轮）`），所有问题收集为内部列表，扫描完成后一次性输出简报。避免"扫描噪声"打断用户注意力。

### 决策 4：可自动修复仅限术语替换

两类问题的处理路径不同：

| 类型 | 选项 |
|------|------|
| 可自动修复（术语替换） | 立即修复 / 跳过 / 标记偏差 / 停止 |
| 不可自动修复（架构决策） | 我来处理 / 跳过 / 标记偏差 / 停止 |

纯术语替换可用 Edit 工具执行；凡涉及设计判断或文件创建的，均不可自动修复，须人工介入。

### 决策 5：govern.md 安装到目标工程

通过 Phase 6 步骤 6.2b 和 `/harness` 步骤 5.5 将 `govern.md` 推送到目标工程，使目标工程的 `/govern` 与 harness 仓库保持同步。
